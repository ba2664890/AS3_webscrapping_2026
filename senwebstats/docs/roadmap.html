01 · Cadre du projet
Contexte & Problématique
Le Sénégal vit une transformation numérique rapide — pénétration internet dépassant 75% en zones urbaines, explosion du mobile money, émergence d'une presse digitale influente. Pourtant, les données fiables sur l'activité web locale restent fragmentées et réservées à ceux qui peuvent payer plusieurs centaines de dollars par mois.

Les outils comme SimilarWeb, Semrush ou Ahrefs offrent des données précieuses à des tarifs inaccessibles aux chercheurs et institutions locales. SenWebStats est né pour construire un observatoire numérique ouvert, autonome et évolutif — capable de collecter, modéliser et visualiser les métriques web des sites sénégalais les plus influents sans aucune dépendance à des plateformes commerciales tierces et sans générer de données fictives.

Principe fondateur : zéro donnée fictive
Chaque valeur affichée dans SenWebStats provient d'une source réelle collectée. Les scores manquants sont imputés par la médiane des mesures réelles disponibles — jamais par une constante arbitraire (exit le fillna(50)). Les domaines référents non disponibles sans AWS Athena sont affichés NULL plutôt que 0 fictif.
02 · Finalités
Objectifs & Spécifications
Axe 1
Cartographie de l'écosystème
Identifier et classer 28 sites sur 5 secteurs. Mesurer leur empreinte via backlinks CommonCrawl, PageRank OPR, performance PageSpeed et intérêt de recherche Google Trends.

Scoring 4 dimensions basé sur données réelles. Trafic estimé modulé par le score Google Trends local (facteur ×0.5 à ×1.5). Pondérations : autorité 45% + qualité 35% + technique 20%.

Blocage SERP — Google, DuckDuckGo, Bing
Les trois moteurs de recherche bloquent les requêtes automatisées depuis réseau professionnel (captcha immédiat, status 202 anomaly-modal, erreur 403 PERMISSION_DENIED sur Google Custom Search). Solution adoptée : modèle de scoring multidimensionnel basé sur les 5 sources alternatives. En V2 : intégration ValueSERP ou SerpApi avec IP résidentielles.
04 · Modèle analytique
Formule de scoring & estimation du trafic
score_autorite (45%)
Autorité de domaine
Open PageRank OPR 0–10 normalisé (60%) + pages indexées CommonCrawl CC normalisées (40%). Source : données réelles collectées via API OPR et CDX.
score_qualite (35%)
Qualité SEO / UX
seo_score × 40% + performance_score × 35% + accessibility_score × 25%. Source : API Google PageSpeed Insights. Valeurs manquantes → médiane des mesures réelles (jamais 50 fixe).
score_technique (20%)
Performance technique
Temps de réponse inversé (40%) + SSL (20%) + sitemap (15%) + word_count normalisé (25%). Source : crawl HTML direct. Valeurs manquantes → médiane réelle.
Modulation du trafic par Google Trends
Le trafic estimé intègre désormais le signal d'intérêt de recherche réel au Sénégal :

trafic_estime = base_traffic × (0.5 + trends_score / 100)

Facteur de modulation : de ×0.5 (site peu recherché) à ×1.5 (pic d'intérêt). Un site avec trends_score=100 (Seneweb) reçoit 1.5× son trafic estimé, tandis qu'un site avec score=0 reçoit 0.5×. Ce facteur utilise les données Google Trends réelles collectées pour le Sénégal (geo=SN) sur les 12 derniers mois.

  • main.py unified : commandes crawl | perf | backlinks | trends | pagerank | full | status

10 · Honnêteté scientifique
Limites identifiées
Limites méthodologiques
  • Absence de positions SERP réelles : trafic estimé depuis score composite. Marge d'erreur ±25–40% vs données SimilarWeb.
  • referring_domains = NULL : non disponible sans AWS Athena sur fichiers WAT CommonCrawl (~$100 de requêtes).
  • Corpus limité : 28 sites leaders. Longue traîne du web sénégalais non couverte.
  • Google Trends relatif : score 0–100 relatif au pic — comparaisons inter-secteurs à interpréter avec précaution.
Limites techniques
  • SPA JavaScript : 12 sites React/Vue rendent du contenu en JS non parsable par BeautifulSoup → title=N/A.
  • PageSpeed partiel : ~13/28 sites mesurés. Rate-limiting Google API non résolu.
  • SQLite non scalable : limites à 10 000+ sites ou accès concurrents.
  • Déploiement local : pas encore accessible en ligne. Streamlit Cloud prévu en V2.
